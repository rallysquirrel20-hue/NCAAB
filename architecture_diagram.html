NCAAB March Madness Betting Dashboard -- Architecture

Source code Data file (parquet/JSON/CSV) External API Config / mapping Frontend component

1 -- DATA PIPELINE (run daily: python ncaab_daily_builder.py)

CONFIG Shared Constants
ncaab_config.py 68 tournament teams, conference map, ESPN name aliases, Odds API name map, hardcoded ESPN IDs, API base URLs, rate limits
espn_team_ids.json Cached ESPN team ID resolution (canonical name -> ESPN numeric ID)
team_name_map.json ESPN display name -> Odds API name fuzzy-match mapping
PHASE 1 Fetch New Games
ncaab_daily_builder.py Scans dates since last run. Skips already-stored event IDs.
ESPN Scoreboard API 1 call per date -> final scores, half scores, home/away, neutral site
ESPN Game Summary API 1 call per event -> boxscore stats (FG, 3PT, FT, rebounds, turnovers), AP rank
Dates converted UTC -> Eastern time so late games land on correct calendar date
|
PHASE 2 Fetch Odds
The Odds API (historical) 2 calls per NEW date -> opening & closing snapshots for spreads + moneylines
Closing line = snapshot at floor(commence_time). Opening = earliest snapshot.
Snapshots cached in odds_cache/*.json to avoid re-fetching
|
PHASE 3 Compute Point-in-Time Stats
ncaab_daily_builder.py Pure math, zero API calls. Always recomputed from scratch (fast).
For each team x game (chronological): snapshot running stats BEFORE the game, then update counters
PIT stats: win%, ATS%, PPG, shooting splits (2PT/3PT/FT), defense, rebounds, turnovers, pace, SOS -- all per overall/home/neutral/away
Opponent PIT stats populated when opponent is one of the 68 tracked teams
|
PHASE 4 Export & Save
ncaab_game_logs.parquet Primary data store. ~4000 rows x 129 columns. All teams, all stats.
ncaab_game_logs.csv CSV export -- tournament teams only (is_tracked=True), subset of columns
ncaab_odds_history.parquet Flattened odds snapshots (~2.5M rows). Appended incrementally from odds_cache/. Per-bookmaker spread/ML prices over time.

2 -- BACKEND API (FastAPI on port 8001)

STARTUP Data Loading & Preprocessing
app/backend/main.py Reads both parquets + ncaab_config.py on startup
Step 1: Load game_logs.parquet -> compute ATS margin, covered, push columns
Step 2: Build team stats timeline (team -> chronological PIT snapshots) for ranking computation
Step 3: Load odds_history.parquet -> match closing spread prices to each game via (team, commence_time). Fills actual juice per bet.
Step 4: Tag tournament rounds -- First Four / R64 / R32 / S16 / E8 / F4 -- by each team's game sequence, not by date
Step 5: Build GAMES_DF -- deduplicate to one row per game (favorite's perspective). For games where tracked team is underdog, flip team/opponent/spread/covered so "team" is always the favorite. Attach underdog spread prices. Verify: favorites + underdogs have same N, bucket units sum to parent.
|
ENDPOINTS REST API
GET /api/games?date= Games for a date with scores, spreads, PIT stats for both teams. Powers the Dashboard.
GET /api/games/{id}/line-movement Spread line movement from odds_history for a specific game. Chart data.
GET /api/espn/scoreboard?date= Proxies ESPN live scoreboard. Frontend polls every 30s for live score updates.
GET /api/strategy/summary Strategy stats (N, record, ATS%, units, ROI) across all timeframes x spread buckets x venue splits. Uses GAMES_DF.
GET /api/strategy/equity-curve Cumulative P&L curve for any strategy filter (side, venue, spread range, timeframe). Uses actual closing prices.
GET /api/strategy/games Game-by-game list for a strategy filter. Sorted newest-first. Shows spread, price, ATS margin, P&L per bet.
POST /api/backtest Custom backtester -- stackable filters on PIT stats (absolute, vs opponent, by margin, by percentile rank). Returns record, ATS%, ROI, p-value, CI, equity curve.
GET /api/dates, /api/teams, /api/stats Metadata endpoints for available dates, team list, and backtestable stat names.

3 -- FRONTEND (React + TypeScript + Vite on port 5174)

SHELL App Structure
src/main.tsx React entry point. Renders App into #root.
src/App.tsx Header with tab navigation (Dashboard | Strategy | Backtester). Exports API_HOST for dynamic backend URL.
src/index.css Solarized Light theme. Monospace fonts. Terminal aesthetic (no border-radius).
|
TAB 1 Dashboard -- GameDashboard.tsx
GameDashboard.tsx Date picker with prev/next/today. Fetches /api/games for historical data.
Game cards: Team scores, opening/closing spread, moneyline, ATS result (COVERED/MISSED). Click to expand -> stat comparison table (Win%, ATS%, PPG, 3PT%, FT%, Def 2P%, OREB, TO, Pace, SOS) with color-coded edge column.
Line movement chart: Embedded lightweight-charts mini-chart per game from /api/games/{id}/line-movement
ESPN Live: Toggle to show ESPN scoreboard. Polls /api/espn/scoreboard every 30 seconds. Red pulsing dot on live games.
TAB 2 Strategy -- StrategySummary.tsx
StrategySummary.tsx Fetches /api/strategy/summary on mount. Timeframe pill selector.
Equity curve: Single lightweight-charts panel. Defaults to All Favorites. Updates when any strategy row is clicked via /api/strategy/equity-curve with matching filter params.
All Games table: Favorites (total + 4 spread buckets: 0.5-2.5, 3-5.5, 6-9.5, 10+) then Underdogs (same buckets). Columns: Strategy, Games, Record, ATS%, Units, ROI, Profitable.
Home/Away/Neutral table: Same bucket structure per venue type (Home Fav, Home Dog, Away Fav, Away Dog, Neutral Fav, Neutral Dog).
Game drill-down: Click any strategy row -> fetches /api/strategy/games -> sortable game list with Date, Favorite, Underdog, Score, Spread, Price, ATS Margin, W/L, P&L, Cumulative P&L.
TAB 3 Backtester -- Backtester.tsx
Backtester.tsx Filter builder UI. Posts to /api/backtest.
Side filter: All / Favorites / Underdogs / Home / Away
Min games: Minimum games played filter
Stackable conditions: Stat > value, Stat > opponent, Stat > opp by margin, Team/Opp rank > percentile. Multiple conditions AND'd together.
Results: Games, Record, ATS%, ROI, p-value (binomial), 95% CI (Wilson). Equity curve chart.

4 -- SUPPORTING TOOLS

ncaab_backtest.py CLI backtester. Runs predefined strategies + differential scan. Independent of the web app -- reads parquet directly.
build_team_name_map.py One-time utility. Samples ESPN + Odds API team names across 18 dates, fuzzy-matches them, outputs team_name_map.json.
ecosystem.config.js PM2 process manager config. Starts ncaab-frontend (Vite on 5174) and ncaab-backend (FastAPI on 8001) together.
